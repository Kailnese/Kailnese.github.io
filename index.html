如何计算time complexity和space complexity

Posted on 2020-01-13 In Data Structure and Algorithm

时间复杂度

时间复杂度不是计算具体计算机运行代码时间, 而是一个估计值而已. 就是大概计算算法执行语句的次数
常见的复杂度有:

  • 常数阶(constant): O(1)
  • 对数(logarithmic): O(log2n)
  • 线性(linear): O(n)
  • 线性对数(linearithmic): O(nlog2n)
  • 平方(quadratic): O(n^2)
  • 立方(cubic): O(n^3)
  • k次方: O(n^k)
  • 指数(exponential): O(2^n)

计算方法:

  1. 选取相对增长最高的项
  2. 最高项系数都化为1
  3. 若是常数的话用O(1)表示

e.g. f(n)=2*n^3+2n+100则O(n)=n^3

空间复杂度

空间复杂度是对一个算法在运行过程中临时占用存储空间大小的量度。
计算方法：

  1. 忽略常数，用O(1)表示
  2. 递归算法的空间复杂度=递归深度N*每次递归所要的辅助空间
  3. 对于单线程来说，递归有运行时堆栈，求的是递归最深的那一次压栈所耗费的空间的个数，因为递归最深的那一次所耗费的空间足以容纳它所有递归过程。

Dijkstar Algorithm

Posted on 2020-01-12 In pathfinding algorithm

如图所示:

创建一个table后, 从上到下依次loop, A是起点, F是终点

每次loop都会选出路径总和最小值, 然后继续往下.

  • 比如在第一次iteration中, A到各个点的值分别为, 0(A), 8(B), 2(C), 5(D), infint(E), infint(F), infint(G), infint(H).
  • 然后看到到C的路程只有2, 则下一个目标就是2了.
  • 第二个iteration中, C到各个点的位置能看出, 到D的距离最短, 则下一个iteration就是D.

每次iteration要记得把之前的距离加起来.

Dijstra的时间复杂度为O(n^2)
